**OAuth 2.0** is a framework in which a **client** obtains limited access to **protected resources** on an HTTP API, usually **on behalf of a resource owner**, without handing the resource owner’s long-lived password to the client.

Any resource can define a set of permissions that can be used to divide the functionality of that resource into smaller chunks. These *permissions* are also called *scopes*.
